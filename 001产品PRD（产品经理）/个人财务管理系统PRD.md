## 一、产品概述

### 1.1 产品背景

现代人日常生活中，消费场景多样化，现金、银行卡、微信、支付宝、信用卡等多种支付方式并存，导致消费记录分散、难以统一管理。同时，小微商户和个体户也迫切需要一款简单、免费、无广告的记账工具来管理日常流水。

### 1.2 产品目标

帮助用户**轻松记录每一笔收支，掌握资金流向，明白消费、清晰经营**。

### 1.3 产品定位

面向**个人用户和小微商户**的**轻量级Web端财务管理工具**，主打**简单、可爱、温暖、无广告**。

> 💗 让记账变成一件温暖又开心的事

### 1.4 核心价值

| 核心价值 | 说明 |
|---------|------|
| **简单** | 3秒完成一笔记录，随时随地打开浏览器使用 |
| **清晰** | 自动归类，图表可视化，账目一目了然 |
| **可爱** | 粉嫩配色+圆润界面，记账也能很治愈 |
| **通用** | 个人/家庭/商户均可使用 |
| **隐私** | 数据存储在本地，多用户数据隔离，保护隐私 |

### 1.5 产品形态

| 项目 | 说明 |
|-----|------|
| **客户端** | Web网页（H5响应式） |
| **访问方式** | 浏览器直接打开，支持手机/平板/电脑 |
| **无需安装** | 即开即用，不占用手机内存 |
| **数据存储** | 浏览器本地存储（LocalStorage）+ 可导出备份 |
| **用户系统** | 本地用户注册/登录，数据按用户隔离 |

## 二、目标用户

### 2.1 用户群体

| 用户类型 | 典型画像 | 核心诉求 |
|---------|---------|---------|
| **个人用户** | 18-45岁上班族 | 记录日常消费、了解支出结构、控制不必要消费 |
| **家庭用户** | 夫妻/情侣共同管理财务 | 共同记账、清晰掌握家庭收支 |
| **小微企业** | 10人以内初创公司/工作室 | 简单记录日常经营流水、与个人账目分开 |
| **个体户** | 餐饮/零售/服务等小老板 | 每日收款记录、简单统计、无需复杂会计功能 |

### 2.2 用户场景

**个人用户场景**：
- 早餐买包子：掏出手机 → 打开网页 → 选择分类 → 输入金额 → 完成（15秒）
- 地铁通勤：扫码付款后，顺手记录交通费用
- 月底复盘：查看各分类支出，分析"钱都花哪儿了"

**商户用户场景**：
- 每日收款：记录每一笔营业收入
- 进货支出：记录货物采购成本
- 日结统计：快速查看今日/本周/本月收支概况

### 2.3 用户痛点

| 痛点 | 描述 |
|-----|------|
| 随手记太复杂 | 现有App功能繁杂，记账门槛高 |
| 不知道花哪了 | 月底复盘时发现"钱不知道怎么就没了" |
| 商户工具太贵 | 专业财务软件年费高、学习成本高 |
| 跨设备同步难 | 手机记的电脑看不到，数据不同步 |
| 广告太多 | 免费App广告轰炸，体验差 |
| 多设备数据互通 | 希望同一账号在不同设备访问同一份数据 |

### 3.2 核心功能详解

#### 3.2.1 用户认证模块（V1.0.1新增）

**功能描述**：支持用户注册和登录，实现多用户数据隔离，保护用户隐私。

**注册功能**：

| 字段 | 类型 | 说明 | 必填 |
|-----|------|-----|-----|
| 用户名 | 文本 | 4-20位字母或数字，唯一 | 是 |
| 密码 | 密码 | 6-20位 | 是 |
| 确认密码 | 密码 | 与密码一致 | 是 |

**登录功能**：

| 字段 | 类型 | 说明 | 必填 |
|-----|------|-----|-----|
| 用户名 | 文本 | 注册时填写的用户名 | 是 |
| 密码 | 密码 | 注册时设置的密码 | 是 |

**交互流程**：

1. 首次打开应用，显示登录/注册引导页面
2. 用户可选择「登录」或「注册新账号」
3. 注册：填写用户名→设置密码→确认密码→完成注册→自动登录
4. 登录：填写用户名→填写密码→登录成功→进入首页
5. 登录后，顶部显示用户名，点击可查看用户菜单

**用户菜单**：
- 个人中心：查看账号信息
- 切换账号：退出当前账号，返回登录页
- 删除账号：清除本地所有数据（需二次确认）

**安全说明**：
- 密码使用SHA-256哈希存储，不明文保存
- 每个用户的数据独立存储在LocalStorage中
- 退出登录后需重新输入密码登录
- 支持「记住密码」功能（可选）

**游客模式**：
- 用户也可选择不注册，直接使用「游客模式」体验
- 游客模式下数据存储在临时空间
- 游客切换到登录用户时，可选择：
  - 「合并数据」：将游客数据合并到账号
  - 「清空重来」：丢弃游客数据

#### 3.2.2 记一笔（核心功能）

**功能描述**：用户最常用的操作入口，要求极简、极速。

**收支类型**：
- 支出：记录花费出去的金额
- 收入：记录收到的金额（商户用户更关注）

**输入字段**：

| 字段 | 类型 | 说明 | 必填 |
|-----|------|-----|-----|
| 类型 | 切换 | 支出 / 收入 | 是 |
| 金额 | 数字 | 支持小数，如"23.50" | 是 |
| 分类 | 选择 | 支出分类或收入分类 | 是 |
| 账户 | 选择 | 从账户列表选择 | 是 |
| 日期 | 日期 | 默认为当天，可修改 | 否 |
| 备注 | 文本 | 描述内容，如"星巴克拿铁" | 否 |

**交互流程**：

1. 用户点击首页「记一笔」按钮（或快捷图标）
2. 进入快捷记账页面（默认展开数字键盘）
3. 选择「支出」或「收入」→ 输入金额 → 选择分类 → 选择账户 → 确认
4. 页面显示「记账成功」提示，自动返回首页

**快捷操作**：
- 首页显示常用分类图标，点击直接记
- 支持快捷键（电脑端）加速输入

---

#### 3.2.3 账单列表

**功能描述**：展示所有记账记录，支持多种筛选方式。

**功能要点**：
- 默认按日期倒序展示（最新在上）
- 支持按日/周/月筛选
- 支持按收支类型筛选（全部/支出/收入）
- 支持按分类筛选
- 支持按账户筛选
- 支持关键词搜索备注
- 每条记录显示：类型图标、金额、分类、账户、备注、日期
- 点击记录可编辑/删除

**操作**：
- 左滑/长按显示删除按钮
- 点击记录弹出编辑抽屉

**空状态**：
> 还没有记账记录，
> 点击下方「记一笔」开始吧～

---

#### 3.2.4 月度统计

**功能描述**：帮助用户了解收支结构。

**展示内容（个人/家庭）**：
- 本月总支出 vs 总收入
- 各分类支出金额排行（柱状图）
- 各分类支出占比（饼图/环形图）
- 与上月对比（如：餐饮 ↑ 15%）
- 每日支出趋势（折线图）

**展示内容（商户）**：
- 今日/本周/本月收入总额
- 收入分类排行（如：商品销售、服务收入）
- 支出分类排行（成本项）
- 净利润计算（收入 - 支出）
- 每日收入趋势（折线图）

**交互**：
- 点击分类可查看该分类下的所有记录明细
- 支持切换月份查看历史数据

#### 3.2.5 账户管理

**功能描述**：管理用户所有的"钱包"或"收款账户"。

**预设账户**：

| 账户名称 | 说明 | 适用用户 |
|---------|------|---------|
| 现金 | 手中的现金 | 全部 |
| 银行卡 | 储蓄卡/信用卡 | 全部 |
| 微信 | 微信支付/收款码 | 全部 |
| 支付宝 | 支付宝/收款码 | 全部 |
| 收钱码 | 商户收款码 | 商户 |
| 其他 | 自定义账户 | 全部 |

**账户功能**：
- 支持新增自定义账户（如：公司账户、客户预存款等）
- 支持修改账户名称
- 支持删除账户（需无关联记录）
- 每个账户显示：名称、当前余额、记录笔数

**余额说明**：
- 系统自动根据收支记录计算各账户余额
- 账户余额仅供参考，以实际为准

#### 3.2.6 预算管理

**功能描述**：设置月度预算，控制消费超支风险。

**功能要点**：
- 设置月度总预算金额
- 可按分类设置子预算（如：餐饮预算2000，购物预算1000）

- 超支后以醒目颜色提示

**适合场景**：
- 个人用户控制"月光"
- 家庭用户规划月度开支
- 商户控制某类成本支出

## 四、数据统计口径

### 4.1 核心指标

| 指标 | 计算方式 | 说明 |
|-----|---------|------|
| 总支出 | Σ支出记录金额 | 当期所有支出之和 |
| 总收入 | Σ收入记录金额 | 当期所有收入之和 |
| 净收益 | 总收入 - 总支出 | 商户关注指标 |
| 结余 | 总收入 - 总支出 | 个人关注指标 |
| 账户余额 | 期初余额 + 收入 - 支出 | 系统自动计算 |

### 4.2 统计周期

| 周期 | 说明 |
|-----|------|
| 今日 | 当天0点至当前时间 |
| 本周 | 本周一至当前日期 |
| 本月 | 本月1日至当前日期 |
| 自定义 | 用户选择起止日期 |

## 六、视觉设计规范

### 6.1 设计风格

产品整体风格为**粉嫩可爱、温暖清新**，给人温馨、轻松愉悦的感觉，让记账不再是枯燥的事情。

| 设计要素 | 风格描述 |
|---------|---------|
| **整体调性** | 温暖、柔和、可爱、治愈 |

| **阴影效果** | 轻柔的投影，不刺眼 |

## 九、竞品分析

### 9.1 主要竞品

| 产品 | 类型 | 优点 | 不足 |
|-----|------|------|------|
| 随手记 | App | 功能全面、用户基数大 | 复杂、付费功能多、广告多 |
| 钱迹 | App | 简洁、无广告、免费 | 功能单一、仅App端 |
| 支付宝记账 | App/小程序 | 整合支付数据 | 仅限支付宝用户、功能有限 |
| 微信记账本 | 小程序 | 微信原生、数据同步 | 仅支出、功能单一 |
| Excel/纸质账本 | 传统 | 灵活、可定制 | 繁琐、无法自动统计 |

### 9.2 差异化方向

| 差异化点 | 说明 |
|---------|------|
| **Web端优先** | 浏览器即用，跨设备访问，无需下载 |
| **个人+商户兼顾** | 同时满足个人记账和小微商户记账需求 |
| **极简体验** | 不做大而全，聚焦核心功能，界面干净 |
| **无广告** | 纯工具定位，界面清爽专注 |
| **本地存储** | 数据存储在浏览器本地，保护隐私 |
| **免费使用** | 基础功能全部免费，不设付费墙 |
| **多用户支持** | 注册登录后数据按用户隔离，支持多设备访问同一账号 |

### 9.3 竞争优势

1. **零门槛**：打开浏览器就能用，不用注册账号（游客模式）
2. **无广告**：纯净体验，不打扰
3. **跨设备**：手机、平板、电脑通用
4. **数据可控**：导出备份自己管理，不用担心平台跑路
5. **多用户**：注册账号后数据同步，多设备无缝使用

### 10.1 技术选型

| 存储 | LocalStorage | 浏览器本地存储 |
| 打包 | Vite | 快速构建 |

### 10.2 数据存储

- **本地存储**：所有数据存储在浏览器LocalStorage
- **容量限制**：大多数浏览器支持5-10MB
- **数据迁移**：支持导出CSV/Excel进行备份和迁移
- **用户隔离**：每个用户的数据使用独立Key存储，实现数据隔离
- **用户表结构**：
```javascript
{
  users: [{
    username: string,      // 用户名
    passwordHash: string,  // SHA-256哈希后的密码
    createdAt: string     // 注册时间
  }],
  currentUser: string     // 当前登录用户名
}
```
